## Root Cause — Stanford 50% Bug

Migration 008 ran:
```sql
UPDATE colleges SET acceptance_rate = COALESCE(acceptance_rate, 0.50)
WHERE sat_25th IS NULL OR sat_75th IS NULL;
```
This set `acceptance_rate = 0.50` (decimal) for any school missing SAT data. The discover page then multiplied `0.50 × 100 = 50`. Fix: remove `×100` from UI and restore true values via SQL.
